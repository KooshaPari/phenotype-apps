## 3. Track specifications

### T1 — L1 Architecture ADR consolidation (2.5 → 3.0)

**Goal:** Consolidate the 47 ADRs across 7 dates into a single cross-referenced index with auto-generated backlinks.

**Deliverables:**
1. `docs/adr/INDEX.md` — single canonical entry point with 47 ADR rows + status + scope
2. `scripts/adr_index_gen.py` — auto-gen script (parses ADR files, builds index)
3. `scripts/adr_backlink_check.py` — verifies every ADR cross-ref points to a real file
4. CI gate: `.github/workflows/adr-lint.yml` — fails PR if backlinks are stale
5. ADR docstring lint: each ADR must have a "## Cross-references" section

**Acceptance:** `adr_index_gen.py` produces 47-row table; `adr_backlink_check.py` exits 0; CI workflow triggers on ADR changes.

### T2 — L44 Performance flamegraph deep-dives (2.0 → 2.5)

**Goal:** Add flamegraph-driven optimization loop for 3 fleet-critical crates (config, tracing, mcp-router).

**Deliverables:**
1. `benchmarks/flamegraph/` — `cargo-flamegraph` integration + scripts
2. `benchmarks/flamegraph/config-baseline.svg` — baseline flamegraph for pheno-config
3. `benchmarks/flamegraph/tracing-baseline.svg` — baseline flamegraph for pheno-tracing
4. `benchmarks/flamegraph/mcp-router-baseline.svg` — baseline flamegraph for pheno-mcp-router
5. `findings/2026-06-22-v20-T2-L44-flamegraph-deep-dives.md` — analysis report
6. Optimization opportunities logged in `findings/2026-06-22-v20-T2-L44-opportunities.md`

**Acceptance:** 3 flamegraph SVGs checked in; opportunities doc identifies ≥ 5 actionable optimizations.

### T3 — L36 Chaos fault-injection framework (2.0 → 2.5)

**Goal:** Build a reusable fault-injection primitive and adopt it in 3 crates (network, fs, async-runtime).

**Deliverables:**
1. `chaos-injection/` (new substrate primitive crate) — `Fault` enum + injection points
2. `chaos-injection/src/lib.rs` — public API
3. `chaos-injection/src/inject.rs` — fault injection at runtime
4. Adoption tests in 3 crates: `pheno-fs`, `pheno-net`, `pheno-async`
5. `findings/2026-06-22-v20-T3-L36-fault-injection-framework.md`

**Acceptance:** `chaos-injection` builds clean; 3 adoption tests pass; framework is reusable.

### T4 — L27 Pact consumer-driven contract tests (2.0 → 2.5)

**Goal:** Add Pact contract tests across 2 federated services (mcp-router ↔ tracing, mcp-router ↔ events).

**Deliverables:**
1. `pact-consumer/` (new test fixture crate) — Pact broker setup + provider stubs
2. Consumer contract: `pheno-mcp-router/pact/contracts/mcp-tracing.json`
3. Consumer contract: `pheno-mcp-router/pact/contracts/mcp-events.json`
4. Provider verification: 2 test files in `pheno-tracing/pact/` + `pheno-events/pact/`
5. `findings/2026-06-22-v20-T4-L27-pact-contract-tests.md`

**Acceptance:** 2 consumer contracts published; 2 provider verifications pass; can-i-deploy check works.

### T5 — L23 `proptest::Arbitrary` impls (2.0 → 2.5)

**Goal:** Add `Arbitrary` derive to 8 of 11 substrate crates so proptest can fuzz them automatically.

**Deliverables:**

4. `findings/2026-06-22-v20-T5-L23-proptest-arbitrary.md`

**Acceptance:** 8 crates have `Arbitrary` impls; 8 new property tests pass; CI workflow runs.
