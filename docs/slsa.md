## What is SLSA L3?

SLSA Build Level 3 means:

1. **Provenance generated by a hardened build platform** — no user-controlled build steps
2. **Build is reproducible** — given same source + build params → identical artifact
3. **Provenance is signed by build platform** — not the user
4. **Tamper-resistant provenance** — included in artifact signature chain

## Phenotype implementation

| Requirement | Implementation | File |
|---|---|---|
| Hardened build platform | GitHub Actions + ephemeral runners | `.github/workflows/ci.yml` |
| Provenance generation | `slsa-framework/slsa-github-generator@v1.10.0` | `.github/workflows/slsa.yml` |
| Signing (OIDC keyless) | `cosign sign-blob --output-signature` | `.github/workflows/supply-chain-signing.yml` |
| Verification | `slsa-verifier` v2.6.0 + cosign `verify-blob` | same |
| Reproducible build | `--locked --frozen` Cargo flags | ci.yml build step |
| Provenance attestation | Auto-uploaded as artifact | `slsa-provenance` artifact |

## Verification command

```bash
# Local
cosign verify-blob \
  --signature=.sigstore/<commit-sha>.sig \
  --certificate=.sigstore/<commit-sha>.pem \
  --certificate-identity-regexp='https://github.com/KooshaPari/.*@refs/heads/main' \
  --certificate-oidc-issuer='https://token.actions.githubusercontent.com' \
  <commit-sha>

# CI
slsa-verifier verify-artifact <artifact> --provenance-path .slsa/provenance.json
```
