## Purpose

This document records the **baseline line/branch/function coverage** for the
FocalPoint Rust workspace as of the Stage 3 QA Hardening milestone. Future
coverage runs are compared against these numbers to detect regressions and to
gate merges.

The measurement framework is:

| Layer | Tool | Recipe |
|-------|------|--------|
| Line/branch/function coverage | `cargo-llvm-cov` 0.6.18 | `just coverage` |
| HTML report | `cargo-llvm-cov` (`--html`) | `just coverage-html` |
| Threshold gate | `cargo-llvm-cov` (`--fail-under-lines 85`) | `just coverage-check` |
| Baseline snapshot | `cargo-llvm-cov` (`--json`) + this file | `just coverage-baseline` |
| CI enforcement | `grade.sh` → `cargo llvm-cov --workspace --fail-under-lines 85` | `just grade` |

## Threshold Gates (DAG Stage 3)

The baseline establishes these enforcement thresholds, codified in
`llvm-cov.toml` and enforced by `just coverage-check` / `grade.sh`:

| Metric | Threshold | Rationale |
|--------|-----------|-----------|
| Lines | **85%** | Matches `grade.sh` `--fail-under-lines 85` |
| Branches | **80%** | Branch coverage lag of 5pp behind lines is conventional |
| Functions | **85%** | Mirrors lines threshold |

Any drop below these thresholds in a subsequent run is a **CI failure** under
DAG Stage 3.

## How to Refresh the Baseline

```bash
# 1. Record a new baseline (rewrites this file + writes JSON)
just coverage-baseline

# 2. Review the human-readable summary
just coverage

# 3. Drill into a crate in the browser
just coverage-html   # open target/llvm-cov/html/index.html

# 4. Enforce the threshold locally before pushing
just coverage-check
```

The baseline is intentionally **regeneratable**: the script overwrites the
totals table above. If a deliberate decision raises or lowers a threshold,
update `llvm-cov.toml` and `just coverage-check` in the same commit so the
gate and the report stay in lockstep.
