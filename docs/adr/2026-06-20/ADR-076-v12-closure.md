# ADR-076: v12 Closure — 71-Pillar P0 Remediation Cycle 1

**Date:** 2026-06-20
**Status:** ACCEPTED
**Wave:** v12 (`chore/v13-71-pillar-cycle-2-p0-2026-06-20`)
**Author:** Manager session 2026-06-20 (forge subagent execution)

## Context

The 71-pillar audit cycle 1 (2026-06-22) revealed 4 pillars (L31, L57, L65, L67) consistently below the 2.0 threshold across the 7 nested fleet repos (`pheno-flags`, `pheno-port-adapter`, `pheno-tracing`, `pheno-errors`, `phenotype-ops`, `PhenoCompose`, `PhenoMCP`). Combined mean for those 4 pillars was **1.21**, dragging the org-wide 71-pillar mean to 1.57. ADR-041 mandates weekly cadence (Mon 09:00 PDT).

## Decision

Wave v12 closed 4 P0 pillars by shipping 1,767 LoC across the monorepo root:

| Pillar | Artifact | Mean cycle 1 → cycle 3 |
|--------|----------|------------------------|
| **L31** CI cache stats | `scripts/cache_stats_wrapper.sh` + workflow + design doc | 1.86 → **2.86** (+1.00) |
| **L57** perf regression | `benchmarks/rust/{Cargo.toml,benches/parse_flag.rs}` + `benchmarks/python/pytest.ini` + `benchmarks/perf-budgets.toml` | 1.00 → **2.57** (+1.57) |
| **L65** SSOT auto-check | `scripts/validate-ssot.sh` + `just validate-ssot` + pre-commit hook | 1.00 → **2.00** (+1.00) |
| **L67** CHANGELOG auto | `cliff.toml` + `.github/workflows/changelog.yml` + convention doc | 1.00 → **1.71** (+0.71) |

Net **+0.53** on the 6-pillar subset mean (2.13 → **2.66**).

## Consequences

### Positive

- All 4 P0 closure tracks shipped; no regressions in other 65 pillars.
- New `just validate-ssot`, `just bench`, `just cache-stats`, `just changelog` targets unify monorepo-root with 7 nested repos.
- Pre-commit `validate-ssot` hook catches future SSOT.md violations at commit time.
- Mission 4 candidate selected: `pheno-config` (Rust, score 11/12); 5-PR gate table ready per ADR-035.
- ADR-015 v2.1 schema bump shipped with idempotent migrator; v2.0 deprecation on schedule for 2026-06-22.

### Negative / Tradeoffs

- `cliff.toml` is at monorepo root only; the 5 fleet adopter repos (pheno, pheno-errors, pheno-flags, pheno-port-adapter, pheno-tracing) do not yet vendor it.
- L65 `validate-ssot.sh` detects but does not fix SSOT cross-ref gaps (1 known: 25 nested `AGENTS.md` files lack SSOT references).
- `CHANGELOG.md` regenerated by git-cliff is in working tree as a side-effect (`M CHANGELOG.md`); not yet committed.

### Carry-over to v13

1. T1-T5: Vendor `cliff.toml` to 5 fleet repos (5 PRs, ~30 min each)
2. T6: Pheno-flake devcontainer template adoption in 5 repos (5 PRs, ~1h each)
3. T7: `ssot-inject` auto-fix script
4. T8: `cache-stats-dashboard` GitHub Pages
5. T9: PR CI `just bench` gate (fail if 1.5x budget exceeded)
6. Mission 4 PR1: `phenotype-config` → Configra migration
