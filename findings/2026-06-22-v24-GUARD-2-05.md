# Guardrail: Pre-commit plaintext secret scanner (GUARD-2-05)

**Date:** 2026-06-22
**Owner:** security circle
**Pillar:** L50 (Security — Vault migration, secret backend)
**Status:** Proposed

## Current state

The v24 T4 plan migrates 3 repos from plaintext env vars to Vault. During the
3-PR cutover (vault client → parallel run → cutover), developers may still
commit `API_KEY=...` strings while testing locally, especially in feature
branches that get force-pushed. `trufflehog` runs in CI nightly, but a secret
merged to `main` between scans lives for up to 24 h. There is no pre-commit
gate; secrets leak via git history even when rolled back.

## Proposed guardrail

Add a `lefthook.yml` pre-commit hook (already canonical per
`phenotype-ops/governance/lefthook.yml`) that runs `pheno-secret-scan` on
staged files. The scanner uses the same regex + entropy heuristic set as
`trufflehog` but is in-process Rust (~5 ms for 100-file diff). Hook blocks
the commit with a clear "this looks like an AWS key, rotate via
`./scripts/rotate-secret.sh`" message. CI gets a *second* scan on the merged
tree (defense in depth).

## Concrete implementation

`pheno-secret-scan` (~200 LOC, reuses `trufflehog` regex set under
`MIT-licensed-patterns.toml`):

```rust
pub fn scan_diff(diff: &str) -> Vec<Finding> {
    let rules = load_rules();
    diff.lines()
        .filter(|l| l.starts_with("+"))
        .flat_map(|l| rules.iter().flat_map(|r| r.match_line(l)))
        .collect()
}
```

`lefthook.yml` (5 LOC):

```yaml
pre-commit:
  commands:
    secret-scan:
      run: cargo run -p pheno-secret-scan -- staged-diff
```
