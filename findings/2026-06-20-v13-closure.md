## v13 Discoveries

1. **cargo-fuzz needs `[workspace]` isolation** — fuzz/Cargo.toml must declare its own workspace root; otherwise it conflicts with the parent workspace
2. **flake.nix format requires exact lock pinning** — flake.lock must commit before nix develop works; rust 1.83, python 3.12, just 1.36 are the canonical pins
3. **SLSA L3 requires github OIDC token** — actions/attest-build needs `permissions: id-token: write`
4. **cosign keyless mode** uses Fulcio + Rekor; requires `cosign sign --yes ghcr.io/...` (no KMS)
5. **chaos-in-CI is opt-in** — only runs on chaos-labelled PRs + weekly cron, to avoid noise on regular PRs
6. **pre-commit hooks work offline** — `local` hooks don't need network access; the `just --evaluate verify` recipe validates justfile structure

## Open Questions / Risks

- **SLSA L3 in fleet adoption**: requires OIDC in every CI workflow. 5 of 7 fleet repos don't have this yet.
- **cosign keyless vs KMS**: keyless is easier but less auditable. Fleet decision pending (ADR candidate).
- **cargo-fuzz nightly required**: stable Rust can't compile `#[no_panic]` annotations. Adds 1 build dependency per fuzzed crate.
