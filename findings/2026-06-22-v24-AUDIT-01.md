# AUDIT-01 — pheno-* fleet inventory vs 47-crate v17 claim (L5)

**Date:** 2026-06-22 (cycle 14, P1 reduction audit)
**Pillar:** L5 (C4 component dependency map, target 1→3)
**Severity:** P1 (claim drift blocks C4 authoring)
**Method:** `ls -d pheno-*` + cross-reference with v17 cycle 7 T2 boundary table (`findings/2026-06-21-v17-L2-module-boundaries.md`).

## Finding

The v17 cycle 7 T2 boundary doc claims **47 pheno-* crates** in the workspace inventory. The on-disk cone for this branch shows **15 pheno-* directories**:

```
pheno-config          pheno-predict
pheno-context         pheno-scaffold-kit
pheno-drift-detector  pheno-secret-scan
pheno-errors          pheno-ssot-template
pheno-framework-lint  pheno-tracing
pheno-llms-txt        pheno-vibecoding-guard
pheno-otel            pheno-worklog-schema
pheno-port-adapter
```

`AGENTS.md:103` corroborates 11 Rust + 4 Python + 1 Go = 16 pheno-* family; sparse-checkout cone excludes the rest. The "47-crate" figure includes `pheno-*` mirror directories inside per-repo workspaces (e.g. `pheno/agileplus/crates/*`), which is correct per the v17 boundary doc convention but does not match the `pheno-*-lib` substrate naming used in ADR-023.

## Impact

The C4 component diagram (T1 deliverable) cannot be authored without a single-source-of-truth inventory. The boundary doc, `AGENTS.md`, and the on-disk cone disagree by ~30 crates. **Pick one convention** before T1 starts.

## Recommendation

- Use **pheno-*-lib** substrate name as the canonical row in the C4 Component diagram.
- Mirror directories under `pheno/*/crates` are **not** separate C4 components; they are workspaces of multi-crate repos. Annotate them with a `parent = <repo>` attribute.
- Update `AGENTS.md:103` to match.
