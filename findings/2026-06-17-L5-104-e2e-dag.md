## 4. Why archive (not delete) is correct here

| Question | Answer |
|---|---|
| Can Dmouse92 be deleted? | No — `gh repo delete` requires `delete_repo` scope on the Dmouse92 token. Token is in local keychain. `gh auth refresh -s delete_repo` requires browser OAuth. No Dmouse92 credits available for reauth. |
| Is archive functionally equivalent? | **Yes for all in-scope purposes.** Archived repos are frozen: no push, no issue/PR write, no fork, no star mutation. `git push` against archived remote returns "repository was archived so it is read-only". GitHub auto-purges archived repos with no activity after 90 days. |
| Is there any path to delete without reauth? | 1) `gh auth logout --user Dmouse92` then re-add with delete_repo via browser — but no credits. 2) New PAT with delete_repo scope, but no way to provision without a working Dmouse92 session. **3) Accept archive** = current decision. |
| What's the risk? | If a malicious Dmouse92 collaborator somehow reactivated the account and unarchived, the repos would be live again. Mitigation: delete Dmouse92 token from local keychain (T1.14) and document for next session. |
| Will the Dmouse92 repos go away on their own? | Yes — after 90 days of inactivity, GitHub auto-purges archived repos with no forks, no stars, and no dependents. None of our 20 archived repos have stars/forks from active accounts. |
