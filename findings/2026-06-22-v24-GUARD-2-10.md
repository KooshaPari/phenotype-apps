# Guardrail: SSOT meta-bundle completeness check (GUARD-2-10)

**Date:** 2026-06-22
**Owner:** governance circle
**Pillar:** L65 (Documentation & SSOT — meta-bundle)
**Status:** Proposed

## Current state

Per `findings/71-pillar-2026-06-20-weekly-2.md`, 0 of 8 substrate repos
ship the SSOT meta-bundle (`CHANGELOG.md` + `VERSION` + `AGENTS.md` +
`llms.txt` + `LICENSE-MIT` + `WORKLOG.md`). The v24 T3 plan adds the
3-file minimal set (CHANGELOG, VERSION, AGENTS.md) per repo. After merge,
no gate prevents a substrate repo from deleting `CHANGELOG.md` in a
"cleanup PR" and silently dropping the meta-bundle. The
`phenotype-registry` refresh (ADR-043) catches the gap but only bi-weekly,
so 2 weeks of unreleased changes can ship without a changelog.

## Proposed guardrail

`pheno-meta-bundle-doctor` binary (~60 LOC) checks 4 invariants per repo:
(1) `CHANGELOG.md` exists and contains an "Unreleased" or current-version
section per Keep a Changelog 1.1, (2) `VERSION` is a single semver line and
matches `Cargo.toml`'s `version`, (3) `AGENTS.md` exists and references the
other 5 meta-bundle files (cross-link check), (4) `WORKLOG.md` exists and
uses v2.1 schema per ADR-025 (the `device:` column). Runs in
`governance-drift.yml` (the 4-workflow baseline) weekly Monday 09:00 PDT
(ADR-041 cadence). PR check on `src/**` changes fails if the doctor reports
missing bundle files.

## Concrete implementation

`scripts/meta-bundle-doctor.sh` (~60 LOC, bash + grep):

```bash
[ -f CHANGELOG.md ] || fail "missing CHANGELOG.md"
grep -q "^## \[Unreleased\]\|^## \[$(cat VERSION)\]" CHANGELOG.md \
  || fail "CHANGELOG missing version section"
[ "$(cat VERSION)" = "$(toml_get version Cargo.toml)" ] \
  || fail "VERSION and Cargo.toml version disagree"
```

CI step (3 LOC added to `governance-drift.yml`):

```yaml
- run: ./scripts/meta-bundle-doctor.sh
```

## Effort

**S** — script + workflow step. ~half-day on MacBook. Backward compatible:
1-cycle informational grace per ADR-041.
