# Finding — Worklog schema v2.1 (11th column `device:`) (L5-103)

**Date:** 2026-06-17
**Task:** L5-103
**ADR:** ADR-025
**Worklog:** `worklogs/L5-103-worklog-v2-1-2026-06-17.json`

## Headline

The worklog schema is bumped from **v2.0 → v2.1** by adding the 11th column `device:` (literal `macbook` or `heavy-runner`). The column is mandatory for new worklogs starting 2026-06-17; v2.0 deprecates 2026-06-22 (5-day migration window). The bump is required by ADR-023 Rule 1 (device-fit gate) — the L6 health audit needs a queryable `device` signal to programmatically detect MacBook-incompatible work.

## Why

- ADR-023 (L5-101) introduced the device-fit gate on 2026-06-15: the MacBook is not a heavy-work device; heavy work must be flagged `device: heavy-runner` so the L6 audit can catch violations.
- Without a first-class column, the `device` signal lived in free-form `notes` (parser-fragile, not greppable, not enum-validated).
- v2.1 is **additive** (1 new column at position 11, after `notes`); the v2.0 10-column shape is preserved for historical rows.

## Schema change

| # | Column | Notes |
|---|---|---|
| 1–10 | (v2.0 columns, unchanged) | `task_id`, `date`, `repo`, `category`, `title`, `commit_sha`, `pr_number`, `status`, `author`, `notes` |
| **11** | **`device`** | `macbook` \| `heavy-runner` (literal string; no other values accepted) |

The other 10 columns are unchanged. Migration is forward-only (v2.0 worklogs remain valid).

## Migration window

- **2026-06-17 (Day 0):** v2.1 accepted; v2.0 still valid; pre-commit hook logs a *warning* for the missing `device` column.
- **2026-06-22 (Day 5):** cutover — pre-commit hook flips from `warn` to `fail` for the missing `device` column; CI rejects PRs with v2.0 worklogs.
- **After 2026-06-22:** new worklogs MUST use v2.1; legacy v2.0 worklogs are grandfathered (no retroactive edit required).

## Allowed values

- `macbook` — local development on the orchestrator MacBook (no heavy build/test runs). The default for ~80% of fleet work.
- `heavy-runner` — work done on a self-hosted runner or dispatched subagent device (the L6 audit's signal for "MacBook-incompatible work").

Empty / unknown is a validator error. The `pheno-worklog-schema` validator (Pydantic) rejects rows where `device` is missing, empty, or not in the allowed set.

- **ADR-025 written:** `docs/adr/2026-06-17/ADR-025-worklog-v2-1-device-column.md` (59 lines).
