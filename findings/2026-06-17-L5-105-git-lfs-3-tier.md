# Finding — Git LFS 3-tier policy (always-track / on-demand / never-track) (L5-105)

**Date:** 2026-06-17
**Task:** L5-105
**ADR:** ADR-027
**Worklog:** `worklogs/L5-105-git-lfs-3-tier-2026-06-17.json`

## Headline

The Phenotype fleet adopts a **3-tier Git LFS policy** for binary assets: (a) **always-track** (mandatory LFS for iOS bundles, native libs, Unity/Unreal assets, disk images); (b) **on-demand** (LFS only when > 5 MB single-file or > 50 files in dir, or build outputs); (c) **never-track** (source code, configs, lockfiles, docs). The policy closes L66 (DX: git LFS guidance) and is the root-cause fix for the 2026-06-17 monorepo strand (3 governance commits + l4-68 pheno-context crate blocked by LFS rejects).

## Why

- L66 has been `⚠ blocked` since the 30-pillar audit. The 71-pillar expansion (ADR-024) kept it blocked; the 2026-06-17 wrap-up stranded 3 governance commits + the l4-68 pheno-context crate (286 LOC) because `git push` to `argis` was rejected (submodules referenced LFS objects not in the local cache; `git config lfs.allowincompletepush true` is not sufficient).
- The 3-tier policy names the asset, the action, and the LFS cache management on heavy-runner devices — replacing ad-hoc "LFS if it's big" with explicit, programmatic rules.
- A single `.gitattributes` per repo is the source of truth; `cat .gitattributes | grep lfs` is the L6 audit check.

## The 3 tiers

### Tier 1 — ALWAYS track (mandatory LFS)

| Asset | Pattern | Why |
|---|---|---|
| **iOS `.xcframework`** | `*.xcframework/** filter=lfs diff=lfs merge=lfs -text` | 100 MB–2 GB per bundle; pushes repos past GitHub's 5 GB soft cap. **This is the asset that blocked the 2026-06-17 monorepo push.** |
| **iOS `.framework` bundles** | `*.framework/** filter=lfs diff=lfs merge=lfs -text` | MB-scale; same reason. |
| **Compiled `.a` / `.dylib` / `.so` static libs** | `*.a filter=lfs …`, `*.dylib filter=…`, `*.so filter=…` | Compiled artifacts; not diff-able. |
| **Unity / Unreal asset bundles** | `*.unitypackage filter=lfs …`, `*.uasset filter=lfs …` | Binary blobs; diffs meaningless. |
| **Disk images (`.dmg`, `.iso`)** | `*.dmg filter=lfs …`, `*.iso filter=…` | Single-file GB-scale. |

### Tier 2 — ON-DEMAND track (LFS only when justified)

Default: not in LFS. Add LFS only if any of: single file > 5 MB, OR directory contains > 50 files of the same type, OR the asset is a build output / vendored dep / 3rd-party SDK.

Examples: Rust/Go release binaries (prefer `gh release upload` over LFS), Python wheels, vendored deps with binaries, test fixtures (images/audio/video), generated docs (`cargo doc` HTML, PDF).

**Rule of thumb:** "Generated artifacts belong in CI release artifacts, not in git." If the file is reproducible from source, it does not belong in git at all (`.gitignore` > LFS).

### Tier 3 — NEVER track (explicit non-LFS)

Source code (`.rs`, `.py`, `.go`, `.ts`, `.swift`), small text configs (`.toml`, `.yaml`, `.json`), lockfiles (`Cargo.lock`, `package-lock.json`, `go.sum`), Markdown / docs source (`.md`), SPDX headers, LICENSE files. The audit flags any violation.

## LFS cache management on heavy-runner devices

Heavy-runner devices (`device: heavy-runner` per ADR-025) must:

1. **Pre-warm LFS on checkout** — `git lfs fetch --all` runs as a setup step before any `cargo test` / `pytest` / `go test` that touches a Tier-1 asset.
2. **Pin LFS object versions in CI** — the CI workflow checks `git lfs ls-files | sha256sum` against a `LFS_LOCKSUM` file (committed in repo root) to detect silent LFS regressions.
3. **Clean LFS on worktree close** — `git worktree remove` runs `git lfs prune --local` to prevent cache bloat.
4. **Quota on heavy-runner** — 50 GB LFS cache ceiling; `git lfs prune --warn-when-above 50G` is the CI guard.

**Local-dev override (MacBook, `device: macbook`):** Tier-1 assets may be checked out **without** LFS if the local task does not need them (sparse-checkout the dir, leave the LFS pointer in place). The build will then fail locally for any task that needs the asset — which is the signal to dispatch to a heavy-runner (ADR-023 Rule 1).
