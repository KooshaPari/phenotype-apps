## 4. Findings

### 4.1 CRITICAL

#### C1 — ADR-038 pattern contract does not match `pheno-port-adapter` actual code

The PR repeatedly claims `pheno-port-adapter` is the **reference impl** of ADR-038 (per `AGENTS.md:12`, `SPEC.md:1,3,5`, `STATUS.md:1,6`, `llms.txt:1,2`, `CHANGELOG.md:18,44`). However, the actual `PortAdapter` trait in `pheno-port-adapter/src/lib.rs:24-29` does **not** match the ADR-038 pattern contract in `docs/adr/2026-06-18/ADR-038-hexagonal-port-adapter-l4-policy.md:32-49`.

**ADR-038 canonical pattern contract** (`docs/adr/2026-06-18/ADR-038-hexagonal-port-adapter-l4-policy.md:34-42`):

```rust
pub trait Port: Send + Sync {
    type Request: Send;
    type Response: Send;
    type Error: std::error::Error + Send + Sync + 'static;
    async fn execute(&self, req: Self::Request) -> Result<Self::Response, Self::Error>;
}
```

**Actual `pheno-port-adapter` code** (`pheno-port-adapter/src/lib.rs:24-29`):

```rust
pub trait PortAdapter: Send + Sync {
    fn name(&self) -> &str;
    fn health(&self) -> Result<(), AdapterError>;
    fn connect(&self, endpoint: &str) -> Result<Connection, AdapterError>;
    fn disconnect(&self) -> Result<(), AdapterError>;
}
```

**Five concrete mismatches:**

| Element | ADR-038 (canonical) | `pheno-port-adapter` (actual) | Match? |
|---|---|---|---|
| Trait name | `Port` | `PortAdapter` | NO |
| Method count | 1 (`execute`) | 4 (`name`, `health`, `connect`, `disconnect`) | NO |
| Method signature | `async fn execute(&self, req: Self::Request)` | sync `fn`, no request param | NO |
| Associated types | `type Request`, `type Response`, `type Error` | none | NO |
| Error type | generic `type Error: std::error::Error` | fixed `AdapterError` enum | NO |

The actual code follows **ADR-014 (predecessor)**, not ADR-038. ADR-014 introduced a `Port` trait + `Adapter` impl shape, and `pheno-port-adapter` is the in-tree implementation of that — but the ADR-038 "canonical substrate interface" decision introduces a new generic + async + associated-types shape that `pheno-port-adapter` does not adopt.

**Impact:** If 19 of 22 pheno-* Rust crates are about to migrate to the "Port trait + Adapter impl shape" per `STATUS.md` and `AGENTS.md`, they will encounter a **mismatch between the documented pattern contract (ADR-038) and the reference impl (ADR-014-era code)**. The 19 ad-hoc crates will not be able to use `pheno-port-adapter` as a copy-and-extend template.

**Suggested fix (in a follow-up PR):**

- **Option A (preferred):** Add a thin compatibility shim — a new `Port` trait (the ADR-038 canonical) in `pheno-port-adapter` that wraps `PortAdapter` via the `execute` pattern. Deprecate the use of `PortAdapter` as the canonical name.
- **Option B:** Update ADR-038 to reflect the actual sync, multi-method, no-associated-types shape that `pheno-port-adapter` uses. Mark the async + associated-types shape as a v0.2.0 follow-up.
- **Option C:** Keep `pheno-port-adapter` as the ADR-014 reference impl (it predates ADR-038), and demote ADR-038's "reference impl" claim to "adoption target reference" — i.e. a new v0.2.0 crate or a major refactor.

The PR's "reference impl" claim is factually wrong and must be reconciled with the actual code before the 19-crate migration wave begins.

#### I2 — `CHANGELOG.md` v2.0/v2.1 deprecation note is misleading

`pheno-port-adapter/CHANGELOG.md:39-40` (under `### Deprecated`):

> `WORKLOG.md` v2.0 schema (10 columns, no `device:` field) — use v2.1; deprecation date 2026-06-22 per ADR-025.

The prior WORKLOG.md in this repo (`pheno-port-adapter/WORKLOG.md` before the diff) was **NOT a 10-column v2.0 schema**. It was an **11-column ad-hoc schema** with the header row `|task_id|date|repo|category|title|commit_sha|pr_number|status|author|device|notes|` and `device` was already in position 10.

The "v2.0 10-column" is the schema that `pheno-worklog-schema` v2.0 defined (per ADR-015), but **this repo's prior schema wasn't v2.0 — it was an even-earlier ad-hoc shape that included a `device` column in a non-canonical position**.

**Suggested fix:** Replace with: "WORKLOG.md prior ad-hoc schema (11 columns, `device` in position 10) — use v2.1; deprecation date 2026-06-22 per ADR-025." This is more accurate for this repo's history.

#### I4 — `llms.txt` "## Optional" includes non-optional content

`pheno-port-adapter/llms.txt:38-46` (the `## Optional` section) lists:

- `LICENSE-MIT` — correct, this is optional per llmstxt.org spec.
- `Cargo.toml` — **not optional**; it's the package manifest.
- `src/lib.rs`, `src/adapters/mod.rs`, `src/adapters/tcp.rs`, `src/adapters/unix.rs` — **not optional**; these are the actual implementation that LLMs reading the `Public API` should understand.

Per the llmstxt.org spec, the `## Optional` section is for **content that an LLM may skip without losing the project's intent** — typically LICENSE, contributing guides, and security policies. Source code is **not** in this category; the spec encourages listing source files under `## Documentation` (with a `> Note: this is a Rust library` warning if needed) or a separate `## Source` section.

**Suggested fix:** Move `Cargo.toml` and `src/*` out of `## Optional`. Either:

- (a) Delete the source-file entries (the LLM can use the `Public API` rustdoc in the AGENTS.md/SPEC.md instead), or
- (b) Add a `## Source` section after `## Documentation` listing the source files with one-line descriptions.
