# AUDIT-12 — Worklog schema v2.0→v2.1 migration status (governance)

**Date:** 2026-06-22
**Domain:** Governance (ADR-015 v2.1)
**Severity:** P1 (v2.0 deprecates 2026-06-22, tomorrow)
**Method:** `grep -l "device:" worklogs/*.json` + `ls worklogs/`.

## Finding

`worklogs/` contains 11 files. 10 are JSONL/markdown worklogs. Of those, only a subset are confirmed migrated to v2.1 schema (with `device:` field) per `AGENTS.md:226` ("4 fleet WORKLOG.md migrated"). The fleet-root `WORKLOG.md` is at the meta-repo root.

Per `AGENTS.md:226`, v2.0 deprecation is 2026-06-22 — **the system date**. Migration status is not directly observable from this audit without reading every worklog JSON for the `device:` key.

## Impact

If migration is incomplete at deprecation date, CI will start rejecting new worklogs (per ADR-015 v2.1). The cycle 14 orchestrator worklog `2026-06-22-v24-cycle-14-71-pillar-p1-orchestrator.json` is the canonical input for cycle 14; it must be v2.1.

## Recommendation

- Read every file under `worklogs/` and verify `device:` field presence.
- If any v2.0 worklog is in active use, migrate it before 2026-06-22 EOD.
- Update `AGENTS.md:226` with current migration count.
