# AUDIT-04 — Dashboard and alert inventory (L14 + L63)

**Date:** 2026-06-22
**Pillars:** L14 (dashboards), L63 (SLO burn-rate alerts)
**Severity:** P1 (L63 directly blocked; L14 panel needs target host)
**Method:** `find . -maxdepth 3 -name "dashboards" -type d` + `find . -maxdepth 4 -iname "*slo*"` + `find . -maxdepth 3 -name "alerts" -type d`.

## Finding

- **Dashboards:** 1 location only — `PhenoObservability/dashboards/`. No `dashboards/` at the meta-repo root.
- **SLO files:** `OmniRoute-b10/ops/slos.yaml` is the only canonical SLO YAML in the cone. `OmniRoute-b10/tests/unit/quota-burn-rate.test.ts` (and 4 sibling OmniRoute-* test mirrors) is the only burn-rate unit test.
- **Alerts directory:** not present at fleet root or in PhenoObservability; alerting lives in `PhenoObservability/alerting/rules.go` (Go source, not Prometheus rules YAML).

## Impact

- **L63 (T5)** cannot land `alerts/slo-burn-rate.yaml` at the fleet root because no such target exists. Authoring requires creating the `alerts/` directory and a Prometheus rules file with 4 multi-window alerts (1h/6h fast burn, 6h/24h/3d slow burn per SRE ch. 5).
- **L14 (T2)** dashboard JSON must target either the per-repo `PhenoObservability/dashboards/` or a new fleet-root `dashboards/`. ADR-063 (SLO/burn-rate policy, cited in plan:63) does not specify which.

## Recommendation

- Adopt fleet-root `dashboards/` + `alerts/` as canonical for new artifacts (matches L63 plan target).
- Migrate `OmniRoute-b10/ops/slos.yaml` content into `findings/slo-definitions.md` (already partially exists at `findings/2026-06-22-v24-slo-definitions.md`) and reference it from `alerts/slo-burn-rate.yaml`.
