## 2. Test code

### 2.1 Mock OpenAI server (httptest)

`mockOpenAI` (lines 30-130) implements the slice of the OpenAI `/v1/chat/completions` contract the harness needs:

- **Method/path:** POST `/v1/chat/completions` returns 405 on the wrong method, 404 on unknown paths.
- **Request body:** parsed as `{model, messages[]}`; missing model is filled with the canned `m.model`.
- **Canned response:** `{"id":"chatcmpl-mock-…","object":"chat.completion","model":…,"choices":[{"index":0,"finish_reason":"stop","message":{"role":"assistant","content":"…"}}],"usage":{"prompt_tokens":N,"completion_tokens":M,"total_tokens":N+M}}`.
- **Failure injection:** `failWithStatus` (502 by default in the error-path test) + `failBody` (raw JSON body for the error envelope).
- **Capture:** every request is appended to `m.requests` with `Method`, `Path`, `Authorization` header, raw body, and `GotAt`. `requestCount()` and `lastRequest(t)` expose the capture for assertions.
- **Lifecycle:** `httptest.NewServer` + `t.Cleanup(m.server.Close)` — no leaked goroutines.

### 2.2 Minimal router (`router` struct in the e2e package)

Defined at lines 160-260. It exists because `internal/router/router.go` (the production-shape router) currently does not build — see V12-T01 §2 and the new bugs in §6 below.

The test-local router:

- Holds `baseURL`, `apiKey`, `*http.Client` (10 s timeout), and a mutex-guarded `[]auditEntry`.
- `Route(ctx, *sdk.Request) (*sdk.Response, error)`:
  1. Builds an OpenAI wire payload from `sdk.Request` (model, messages, tools, Params pass-through).
  2. POSTs to `baseURL + "/v1/chat/completions"` with `Authorization: Bearer <apiKey>` and `Content-Type: application/json`.
  3. Decodes the response into `sdk.Response`, populating `Provider`, `Model`, `Content`, `Usage`.
  4. On non-200 status returns `*sdk.Error{Code, Message}` (wraps the raw body verbatim — matches OpenAI's `{error:{message,…}}` envelope).
  5. On every path (success or error) writes one `auditEntry` with `At`, `RequestID`, `Provider`, `Model`, `Status`, `LatencyMS`, `TokensIn`, `TokensOut`, `Err`.

When the production router is repaired, the only call site that needs to change is the `newRouter` helper in `e2e/e2e_test.go:165` — the test bodies stay the same.

### 2.3 OTel probe (no-op unless `PHENOTYPE_E2E_OTEL=1`)

`otelSpan{Name, Start, End, Attributes}` (lines 290-310) is the minimal span record the harness emits. The real OTel SDK (`go.opentelemetry.io/otel`) is **not** wired — `go.mod` has zero dependencies — so this struct lets the harness record span-shaped data without forcing a dependency. When OTel is added in V12 (likely T17 or T18), this can be replaced with `oteltrace.Span` without changing test assertions.

`otelEnabled()` returns `strings.EqualFold(os.Getenv("PHENOTYPE_E2E_OTEL"), "1")` — off by default, opt-in via env var. The default-off design keeps the harness self-contained: a fresh checkout with no deps installed can still pass `go test ./e2e/` green.

## 5. Bug found and NOT fixed in scope

### 5.1 `go build ./...` still fails — new root cause documented

V12-T01 §2 captured the **symptom** (`Plugin redeclared in this block`, 6 undefined-method errors). That was the state on 2026-06-21 04:23 PDT. **Between V12-T01 and V12-T16, another process mutated the spike and added new files**, changing the failure mode. The new state (2026-06-21 captured by V12-T16):

```
$ go clean -testcache && go build ./...
sdk/plugin.go:20:2: package phenotype-router/internal/sdktypes is not in std
   (/opt/homebrew/Cellar/go/1.26.2/libexec/src/phenotype-router/internal/sdktypes)
$ echo $?
1
```

**Root cause (new):** `sdk/plugin.go` (top-level — note: NOT `internal/sdk/plugin.go`; this is a different file at the module root) imports `"phenotype-router/internal/sdktypes"`. That import path is malformed — the module is `github.com/KooshaPari/phenotype-router` (declared in `go.mod:1`), so the correct path is `"github.com/KooshaPari/phenotype-router/internal/sdktypes"`. Without the host prefix Go resolves the import as a stdlib lookup and fails because no stdlib package at that path exists.

There is also a **second, related problem**: the existence of two `sdk` packages — `sdk/` (top-level, declared as `package sdk` at `sdk/plugin.go:15`) and `internal/sdk/` (also `package sdk`) — creates a case-insensitive import collision the moment anything imports both. The previous build failure masked this; the new build failure surfaces it.

**Recommendation (not actioned):** the ADR-052-aligned `sdk/plugin.go` should be moved into `internal/sdk/plugin.go` (or its sibling types moved out to `internal/sdktypes/types.go` and the top-level `sdk/` deleted). The first option is preferred because the rest of the spike (`internal/router/router.go`, `internal/plugins/*`) already imports `github.com/KooshaPari/phenotype-router/internal/sdk`.

## 6. Why V12-T16 was scoped to `e2e/` only

V12-T01 promoted (and held) the spike to a production repo with the explicit unblock criterion **`builds and tests pass`**. That criterion still fails for the broader spike. V12-T16 was scoped narrowly to the E2E harness, which is independently shippable and does not require the production router to build — the test-local router in `e2e_test.go` is a self-contained substitute. The harness is forward-compatible with the production router: when `internal/router/router.go` is repaired, only the `newRouter` helper at `e2e/e2e_test.go:165` needs to change; the four test bodies are agnostic to which router implementation they exercise.

This is the same "verify what you can; ship what's green; document the rest" pattern V12-T01 used.
