## 5. Cross-cutting findings (synthesized)

### 5.1 Polyglot pattern (Rust core + Python wrapper)
The Phenotype org is converging on this pattern:
- `pheno-config` (Rust) + `phenotype-python-sdk/phenotype-config` (Python)
- `sharecli` (Rust) + `thegent-sharecli` (Python)
- `pheno-profiling` (Python core, Rust `pprof-rs` bridge) — proposed
- `pheno-otel` (Rust) + `ObservabilityKit/performance_kit` (Python) — existing

**Common interface traits:**
- Rust: traits with object-safety, minimal state
- Python: Pydantic BaseSettings or dataclass

### 5.2 Field-name parity
**Gap:** Rust and Python config crates don't share field names. Rust has `url/port/log_level/db_path/feature_flags`; Python has `environment/debug/service_name`. **Fix:** Add missing fields to both sides (PR-6, PR-9 in subagent B's plan).

### 5.3 Hexagonal architecture
- Settly is hexagonal (domain/application/adapters/infrastructure) but oversold — pulls 14 deps for ~1,900 LoC of which only 3 adapters are real implementations.
- pheno-config and phenoShared/config-core are **not** hexagonal — they're flat library code with fewer deps. **The hexagonal pattern is overkill for these use cases.**

### 5.4 SOTA patterns observed
- **Trait-based, object-safe configs** (phenoShared/config-core's `ConfigLoader` + `ValidateConfig`)
- **Numeric priority** (u8 0-100) over named enum (Settly's 6 named levels)
- **Deep-merge semantics** over 4-strategy enum (Settly's `Override/Underride/DeepMerge/AppendArrays`)
- **Format auto-detection** from extension (vs Settly's 3 separate parser structs)
- **TypeScript-style `__all__` declaration** for public API (cheap-llm-mcp)
- **Closure-based regression tests** for public surface (cheap-llm-mcp `test_public_api.py`)
