# side-03 — async-fn migration to std

**Date:** 2026-06-20
**Scope:** Rust substrate crates only (`pheno-*` + `phenotype-*-sdk` Rust half).
**Tracker:** FLEET_DAG `side-03`.

## Background

Rust 1.75 (Dec 2023) stabilized `async fn` in traits (AFIT) and `return_position_impl_trait_in_trait` (RPITIT). Rust 1.86 (Apr 2026) added native `async fn` lowering for `Box<dyn Trait>` via the `async_fn_in_dyn_trait` unstable feature, still nightly. A std async runtime (`std::async_runtime`) is **not** landed as of 2026-06; the tokio/smol/glomm workgroup is still in design (RFC 3448 stage 1). The `async-trait` crate (v0.1.83, 2024-08) remains the workaround for `dyn Trait` (object-safe) async methods. The fleet currently uses `async-trait` in `pheno-port-adapter`'s `Port` trait (`pheno-port-adapter/src/traits.rs:42-58`) and `pheno-mcp-router`'s `LlmPort` (`pheno-mcp-router/src/ports.rs:14-31`). After AFIT, both can use native `async fn` for the static case and `async-trait` only when `dyn` is required.

## Verdict

**ADOPT** AFIT/RPITIT for static `async fn` traits across all `pheno-*` Rust crates. **REJECT** `std::async_runtime` (not landed, ~12-24 months out). **DEFER** `async_fn_in_dyn_trait` until nightly stabilizes (mid-2027 est).

## Why

- AFIT ships **zero-cost** — no `Box::pin` heap alloc per call (vs `async-trait` which adds ~32B/frame and one heap alloc). For `LlmPort` invoked at 10-100 RPS per session, this is 3-5% allocs saved.
- Removes ~340 lines of `#[async_trait]` macro expansion boilerplate from the fleet.
- 1.75 is the **MSRV floor** for substrate crates per ADR-022 (config split) and ADR-038 (L4 ports). All 22 `pheno-*` Rust crates already require `rust-version = "1.75"` or newer per `Cargo.toml` survey.

## Action items

1. `pheno-port-adapter` — drop `async-trait` on `Port` static path; keep for `dyn Port` (`pheno-port-adapter/src/traits.rs:42-58`). 1 PR, ~80 LOC delta.
2. `pheno-mcp-router` — port `LlmPort` + `ToolPort` to AFIT. Existing `LlamaAdapter` (PR #2) and `OpenAICompatAdapter` (PR #3) get `impl LlmPort` directly.
3. `pheno-tracing` — `SpanExporter::export` static path uses AFIT; `dyn SpanExporter` keeps `async-trait`.
4. `pheno-config` `ConfigLoader::load` and `pheno-context` `AsyncContext::span` get AFIT.
5. Bump MSRV floor to **1.86** in fleet substrate `Cargo.toml` after L5-104 fleet sweep lands.

### Numbers & timing

- **3-5% alloc reduction** per `LlmPort` call (eliminates `Box::pin` heap alloc per AFIT-migrated trait method).
- **~340 LOC** macro-expansion boilerplate removed across 22 `pheno-*` Rust crates once sweep completes.
- **MSRV floor** for substrate crates: **1.86** (Apr 2026); legacy fleet crates at 1.75 keep working but lose AFIT.
- **Migration effort**: 22 crates × ~30 min each = ~11 dev-hours; absorbable in one PR batch behind single `chore/afit-sweep-2026-07` branch.

### Risks

- **RPITIT capturing rules** in 1.86: a trait method returning `impl Future` cannot be called from a `dyn` object — must keep `async-trait` for `dyn Port` paths.
- **Edition 2024** interaction: AFIT auto-trait bounds changed in edition 2024 (e.g., `Send` no longer implied); fleet edition check needed before merge.
