## 2. Threat findings

Format: `| threat | location:line | severity | mitigation |`. Severity uses P0/P1/P2/P3 per AGENTS.md convention (P0 = data loss / RCE / cred leak, P1 = DoS / privilege escalation, P2 = defense-in-depth gap, P3 = informational / pattern hazard).

| # | Threat | Location:line | Severity | Mitigation |
|---|---|---|---|---|
| F1 | **Unbounded response body read** — router does `io.ReadAll(resp.Body)` with no `io.LimitReader` cap; a misbehaving provider returning a multi-GB body OOMs the router process | `spikes/go/phenotype-router/e2e/e2e_test.go:281` | **P2** | Wrap with `io.LimitReader(resp.Body, MaxResponseBytes)` (e.g. 16 MiB). On truncation, return `*sdk.Error{Code: 413, Message: "response too large"}` and audit. |
| F2 | **Unbounded inbound body read** — mock handler does `io.ReadAll(r.Body)` without `http.MaxBytesReader`; test-only but copy-paste hazard for the production router (per package doc: when `internal/router` is repaired, `newRouter` will be retargeted) | `spikes/go/phenotype-router/e2e/e2e_test.go:89` | **P3** | When porting `handle` pattern to prod router: wrap inbound `r.Body` with `http.MaxBytesReader(w, r.Body, MaxRequestBytes)`. |
| F3 | **Authorization header captured verbatim** — `capturedRequest.Authorization` stores the raw `Bearer <key>` value; if any future test logs `lastRequest()` (line 479), the secret is written to stdout/test report | `spikes/go/phenotype-router/e2e/e2e_test.go:67-68, 94` | **P3** | Redact in the captured struct: store only `authScheme` ("Bearer" / "ApiKey") and a SHA-256 prefix of the token (first 8 hex chars). Required for prod parity once `internal/router` lands. |
| F4 | **Upstream error body stored in audit log without redaction** — `apiErr.Message = string(respBody)` at line 283 captures the entire upstream error response verbatim; if upstream echoes the request (including `Authorization`), the audit log leaks the key | `spikes/go/phenotype-router/e2e/e2e_test.go:283, 287` | **P2** | Cap `respBody` to 4 KiB before storage; strip `Authorization` / `api_key` / `x-api-key` substrings via a redaction helper before assigning to `apiErr.Message` and `auditEntry.Err`. |
| F5 | **Raw `err.Error()` in audit log** — `json.Marshal`, `http.NewRequestWithContext`, `http.Client.Do`, `json.Unmarshal` errors concatenated directly into audit fields | `spikes/go/phenotype-router/e2e/e2e_test.go:253, 264, 276, 310` | **P3** | These stdlib errors don't carry user secrets, so P3; document that future audit-fed structured logs must apply the same redaction helper as F4. |
| F6 | **No `http.MaxBytesReader` on the request envelope** in the router's `Route()` path — `r.http.Do(httpReq)` sends a body of unbounded size (`req.Params` map iteration at line 245-247 has no cap) | `spikes/go/phenotype-router/e2e/e2e_test.go:230-279` | **P2** | After `json.Marshal`, assert `len(body) <= MaxRequestBytes`; otherwise short-circuit with `413 Request Entity Too Large`. |
| F7 | **`apiKey` passed by string parameter** — no secret-handling abstraction; future prod router will be tempted to log `r.apiKey` directly | `spikes/go/phenotype-router/e2e/e2e_test.go:195, 204-209` | **P3** | Use a `Credential` type with `String()` redacted by default (`Credential: ******`); required for F3/F4 parity in `internal/router`. |
| F8 | **Test-only `apiKey` literals** — strings like `"test-key-not-real"` and `"k"` in test bodies could be mistaken for real keys if copy-pasted into prod | `spikes/go/phenotype-router/e2e/e2e_test.go:409, 540, 581` | **P3** | Add `//nolint:gosec.G101` to test helper with rationale comment, or use `t.Setenv("PHENOTYPE_TEST_KEY", "test-key-not-real")` and read via env. |
| F9 | **No plugin-name validation at registration time** — production plugins return hardcoded names (`intelligent-router`, `tool-router`); if the registry ever accepts user-supplied names, no alphanumeric-only check exists | `plugins/intelligentrouter/router.go:208-210`, `plugins/toolrouter/plugin.go:74-77` | **P3** | Add `ValidatePluginName(name string) error` helper using `regexp.MustCompile(\`^[a-z][a-z0-9-]{0,62}$\`)`; reject `..`, `/`, `\`, whitespace. Apply at registry insertion. (This is upstream of the spike but in the same router architecture.) |
| F10 | **SLM client URLs hardcoded to localhost** — `http://127.0.0.1:8008`, `http://localhost:9001/9002`; if misconfigured in prod to a public address with no auth, anyone on the network can drive the SLM router | `plugins/intelligentrouter/router.go:101-104` | **P2** | Default to `127.0.0.1` only; add startup assertion: `if !isLoopback(host) { return error }`. Allow override via `PHENOTYPE_ROUTER_SLM_URL` env with explicit warning log. |
| F11 | **No `tls.Config` / no `InsecureSkipVerify`** — confirmed via `grep -E 'tls\.Config\|InsecureSkipVerify'` returning empty; default Go `crypto/tls` is used, which is correct (no MITM bypass) | (none — absence is positive) | **P3 (positive finding)** | Document in `AGENTS.md` security section: "router spike ships with no TLS bypass; future `internal/router` MUST NOT introduce `InsecureSkipVerify`." Add to CI grep gate: `! grep -rn 'InsecureSkipVerify' plugins/ spikes/`. |
| F12 | **`context.Background()` in test bodies** — tests never exercise cancellation; production router code correctly propagates `ctx` (`e2e_test.go:258` via `http.NewRequestWithContext`) so this is a test-only artifact | `spikes/go/phenotype-router/e2e/e2e_test.go:446, 451, 542, 584` | **P3** | Add a `TestE2E_ContextCancellation` case: pass `context.WithTimeout(ctx, 1*time.Nanosecond)` and assert error path; locks in the cancellation contract. |
