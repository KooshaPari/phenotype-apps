# AUDIT-15 — CI workflow currency check (governance)

**Date:** 2026-06-22
**Domain:** Governance + DevX (DX L28-L37)
**Severity:** P2
**Method:** `ls .github/workflows/` + look for `pheno-ci-templates` references in workflows.

## Finding

The meta-repo root has `.github/` (per file_list). `AGENTS.md:107` references `pheno-ci-templates` ("`pheno-ci-templates` runs the test matrix, coverage gate, OTLP smoke test"). Workflow currency depends on whether the meta-repo's CI uses the canonical templates.

A full enumeration of `.github/workflows/*.yml` was not performed in this audit (would require cone listing of `.github/` subdirs beyond what file_list provides). The fleet has 70 YAML files in total (per `workspace_extensions` count), of which `pheno-ci-templates` references are an unknown subset.

## Impact

If meta-repo CI diverges from `pheno-ci-templates`, the 80%/70%/60% coverage gate per ADR-040 may not be enforced. CI drift is a P2 not P1 because tests still run; the gate is at risk of being silently bypassed.

## Recommendation

- Read every workflow file in `.github/workflows/` and grep for `pheno-ci-templates` references.
- If a workflow does not reference the canonical templates, file a tracking issue per ADR-042B (substrate quality bar).
- Add a CI lint step that fails if any workflow runs `cargo test` without `cargo-llvm-cov` (or equivalent) for coverage capture.
