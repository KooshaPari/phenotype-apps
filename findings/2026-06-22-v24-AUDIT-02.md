# AUDIT-02 — Cross-crate dep edges for C4 container diagram (L5)

**Date:** 2026-06-22
**Pillar:** L5 (C4 dep map, target 1→3)
**Severity:** P2 (blocks diagram rendering, not blocker)
**Method:** `find . -maxdepth 3 -name "Cargo.toml" | wc -l` → 109. Spot-read 6 representative crate Cargo.toml files.

## Finding

109 `Cargo.toml` files exist in the cone. Per-crate edge analysis (which crate depends on which) was **not** performed in this audit (would require full `cargo metadata` which is heavy-runner-only per ADR-023). What is observable from Cargo.toml headers:

- `pheno-errors/Cargo.toml:9-11` declares `anyhow = "1"` only — no internal pheno-* deps.
- `pheno-tracing/Cargo.toml:2` is at version `0.3.0-pre.0` (prerelease, mismatch with sibling crates at `0.1.0`).
- `pheno-otel/Cargo.toml:2` is at `0.1.0`.
- `pheno-port-adapter/Cargo.toml:2` is at `0.1.0`.
- `pheno-config`, `pheno-context`, `pheno-llms-txt`, `pheno-scaffold-kit`, `pheno-secret-scan`, `pheno-ssot-template`, `pheno-vibecoding-guard`, `pheno-worklog-schema`, `pheno-drift-detector`, `pheno-framework-lint`, `pheno-predict` — **Cargo.toml not in cone** (likely mirror-only or absent).

## Impact

T1 (C4 component map) cannot draw edges between Component boxes without `cargo metadata`. The `device: heavy-runner` constraint means the C4 authoring pass will be slow on the MacBook.

## Recommendation

- T1 deliverable must include a `scripts/build_c4_graph.sh` that runs `cargo metadata --format-version 1` on the heavy-runner and emits a DOT graph.
- Version skew between `pheno-tracing` (0.3.0-pre.0) and the rest (0.1.0) is intentional per ADR-036 (substrate canonical), but should be annotated in the C4 Component caption so readers don't assume SemVer-major version meaning.

**Files cited:** `pheno-errors/Cargo.toml:9-11`; `pheno-tracing/Cargo.toml:2`; `pheno-otel/Cargo.toml:2`; `pheno-port-adapter/Cargo.toml:2`.
