# L12 — Documentation & Single Source of Truth (SSOT)

## Scope
Covers README/CONTRIBUTING/AGENTS.md/SPEC.md/SSOT.md presence + freshness, API doc generation (cargo doc, sphinx, mkdocs, VitePress), single-source-of-truth invariants, and cross-references between governance documents across the Phenotype bloc (AgilePlus + thegent + Tracely + Tracera + satellite repos, ~76 crates).

## SOTA 2026
SOTA 2026 documentation practice (per `repos/AUDIT_BLOC_VS_2026_SOTA.md` §Governance + §Documentation, and `docs/SPEC_DOCUMENTATION_SYSTEM.md`):
- **Diátaxis structure** (Tutorials / How-to / Reference / Explanation) with VitePress sidebar (HeliosLab, thegent).
- **docs.rs** auto-publish for every public crate, enforced via `documentation = "https://docs.rs/<crate>"` in `Cargo.toml`.
- **SSOT.md** precedence table at repo root listing each cross-cutting domain and the canonical file that owns it.
- **AGENTS.md / CLAUDE.md** pair at root, with `AGENTS.md` carrying operating-loop rules and `CLAUDE.md` carrying project-specific overrides.
- **SPEC.md** for behavior contracts; **ADR.md** for reversible architectural decisions; **PRD.md** + **PLAN.md** + **FUNCTIONAL_REQUIREMENTS.md** required at root for greenfield (per `docs/SPEC_DOCUMENTATION_SYSTEM.md`).
- **Auto-indexed docs** (`docs/index.md` regenerated by script; thegent, Tracely, Tracera all do this).
- **Cross-references** between docs (links between SPEC ↔ ADR ↔ FUNCTIONAL_REQUIREMENTS ↔ test tracker) — minimum: every ADR must reference the SPEC clause it implements.

## Recommendations

1. Adopt the **HeliosLab SSOT.md template** as the bloc standard; ensure every repo has one (3 of 4 bloc repos missing).
2. Add `docs:` Justfile/Taskfile recipe to every repo that ships a Cargo workspace (4 of 4 missing this).
3. Polyglot repos (Tracera) need a stack-detected docs recipe similar to the existing `coverage:` recipe at `repos/Tracera/justfile:69-83`.
4. Tracely's `CONTRIBUTING.md` should be expanded to match AgilePlus's 12-section structure; current 24 lines is below bloc median.
5. Tracera's `SSOT.md` should be refactored: precedence table on top, recovered index as appendix; the obsolete checkpoint files (`.PHASE_5_*`, `.BLOCKER_FIX_INSTRUCTIONS.md`) should be moved to `docs/.archive/` per `DOCUMENTATION_TAXONOMY.md`.
6. Make `pheno-ssot-template` (currently a sub-repo) the canonical scaffold for new repos; require a populated `SSOT.md` in PR-template.
7. Cross-reference test: add a CI hook that fails when an `ADR-NNN-*.md` exists without a `<!-- Implements: SPEC-NN §X -->` header.
8. Auto-generate `docs/index.md` and require it to enumerate at minimum: SPEC.md, ADR.md, PRD.md (if present), AGENTS.md, CLAUDE.md, and the per-feature spec dir. The Tracely `docs/index.md:7-13` is too thin to satisfy this.
