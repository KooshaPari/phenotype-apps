## 1. Executive Summary

BytePort is a hybrid-language, self-hosted IaC deployment + portfolio platform. The canonical shipping stack is **Go 1.25 (backend)**, **SvelteKit 5 + TypeScript (frontend)**, and **Rust / Tauri 2 (desktop shell)**. The repository is actively developed but carries significant technical debt: outdated documentation, duplicate CI configuration, build timeouts, and a large number of stale branches. The project has good governance tooling (cargo-deny, cargo-audit, CodeQL, secret scanning, OpenSSF Scorecard) but uneven test coverage (strong Go tests, almost no Rust/TS tests).

## 10. Stabilization Plan

### P0 — Critical (Do before next release)

1. **Remove committed secrets / binaries from repository**
   - Add `backend/database.db`, `crt.pem`, `.DS_Store` to `.gitignore`.
   - Run `git filter-branch` or `git-filter-repo` to purge `database.db` and `crt.pem` from history (they contain sensitive data).
   - Rotate any credentials that may have been in `crt.pem`.

2. **Fix duplicate CI configurations**
   - Merge `ci.yaml` (untracked) into `ci.yml` or delete it.
   - Deduplicate `secret-scan.yml`, `secrets-scan.yml`, `trufflehog.yml` into one canonical workflow.
   - Ensure all workflow actions are pinned to full SHA (fix `ci.yaml` and `secret-scan.yml`).

3. **Fix STATUS.md and README.md accuracy**
   - Rewrite `STATUS.md` to reflect the actual project (Go + SvelteKit + Tauri IaC platform).
   - Remove the retired manifesto (lines 75–707 of README.md) or move it to `docs/legacy-manifesto.md`.
   - Keep the canonical stack description (lines 16–44) and quickstart.

### P1 — High (Do within 2 weeks)

4. **Expand test coverage**
   - Add at least 5 unit tests for `frontend/web/src/` using vitest.
   - Add Rust unit tests for `crates/byteport-transport` and `frontend/web/src-tauri/src/`.
   - Add one integration test that spins up the Go backend and hits a health endpoint.
   - Integrate coverage reporting into CI (Codecov or coverage artifact upload).

5. **Add frontend and Rust CI jobs**
   - Extend `ci.yml` or `go-ci.yml` to include `npm run check` and `cargo test --no-run`.
   - Add a `frontend-ci.yml` workflow for `npm run lint`, `npm run check`, and `npm run build`.
   - Add a `rust-ci.yml` workflow for `cargo clippy`, `cargo test --no-run`, `cargo fmt --check`.

6. **Clean up stale branches**
   - Review branches older than 2026-06-01. For each:
     - If merged → delete.
     - If unmerged but superseded → delete.
     - If contains security fixes not on `main` → cherry-pick and then delete.
   - Target: < 20 total branches.

7. **Parameterize `./start` scripts**
   - Remove hardcoded path `~/temp-PRODVERCEL/Rust/webApp/byte_port` from `start` and `start.bat`.
   - Use relative paths or environment variables (`$BYTEPORT_ROOT`).

### P2 — Medium (Do within 1 month)

8. **Improve documentation**
   - Expand `ARCHITECTURE.md` with persistence boundaries, release flow, and packaging details.
   - Fix `CONTRIBUTING.md` to reference `yarn` instead of `pnpm`, and add a `bootstrap` task to `Taskfile.yml` if it is referenced.
   - Add a `README-Windows.md` cross-reference to the main README.

9. **Address build timeouts**
   - Profile `cargo check` / `cargo build` to identify the slow dependency (likely Tauri + native compilation).
   - Consider caching strategies in CI (`Swatinem/rust-cache`, `sccache`).
   - If the Tauri shell is not critical for all PRs, make the Rust build optional or split into a separate workflow.

10. **Add pre-commit hook installation**
    - Add `task bootstrap` or `just bootstrap` that installs `pre-commit` hooks and downloads dependencies.
    - Document the one-liner setup in README.

11. **Clean up worktrees**
    - After merging the active feature branches (`chore/l4-64-byteport-hex-2026-06-11`, `chore/SD4-2026-06-12`, etc.), prune worktrees.

### P3 — Low (Backlog)

12. **Evaluate Go 1.25 toolchain availability**
    - Go 1.25 is listed in `go.mod` but may not be generally available. Verify with `go version` and CI runners.
    - If 1.25 is prerelease, pin to a stable 1.24 or ensure CI images support it.

13. **Review RUSTSEC ignores**
    - 17 advisories are ignored in `deny.toml`. Set a calendar reminder to re-evaluate quarterly as Tauri evolves.
