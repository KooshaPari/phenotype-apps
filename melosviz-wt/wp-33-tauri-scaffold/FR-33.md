# FR-33: Tauri: scaffold (src-tauri/Cargo.toml + tauri.conf.json)

**File scope:** `desktop/src-tauri/`

## Functional Requirement

The Tauri app MUST scaffold successfully and produce platform-specific bundles
when `cargo tauri build` is run.

## Acceptance Criteria

- `cargo tauri build` produces `Melosviz.app` on macOS, `Melosviz.exe` on
  Windows, and `.msi`/ `.dmg` installers for both.
- `tauri.conf.json` declares identifier `ai.phenotype.melosviz` (unique per
  reverse-DNS convention).
- `capabilities/default.json` exposes minimum viable permission set
  (fs:default + dialog:default + core:* defaults).
- Lib + bin split: `main.rs` calls `melosviz_lib::run()` per Tauri 2 convention.

## Implementation Status

Scaffold complete. Icons directory is empty placeholder (real PNG/ICNS generated
via `cargo tauri icon path/to/icon.png`).
